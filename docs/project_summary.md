# mapred 项目结论分析与修正

通过阅读和分析代码，这个 mapred（Local MapReduce Program）项目是一个用 C 语言编写的本地多进程任务并发分发与执行工具。它的核心思想是借鉴 MapReduce 中的 Map 阶段，将标准输入（或文件输入）的数据按行读取，并发地分发给多个后台子进程（即 mapper 脚本或命令）进行处理，然后再将各个子进程的输出统一收集并输出到标准输出。

具体来说，它实现了以下几个核心功能模块：

## 1. 命令行参数解析与输入重定向 (main.c)

* **支持 `-m (--mapper)` 参数**：指定每个子进程需要执行的 Shell 命令（例如 `php test.php`）。
* **支持 `-c (--count)` 参数**：指定后台并发执行的子进程数量（默认是 2 个）。
* **输入重定向**：支持从指定的输入文件读取数据，如果未指定文件则默认从标准输入（STDIN）读取数据（通过 `dup2` 实现重定向）。

## 2. 多进程并发执行 (os.c)

* 通过 `spawn_process` 函数，利用 `fork` 和 `execl` 启动多个子进程。
* 每个子进程都会通过系统默认 Shell (`/bin/sh -c`) 执行指定的 mapper 命令。
* 在父子进程之间创建两组匿名管道（Pipes），用于重定向子进程的 STDIN 和 STDOUT，从而实现父进程向子进程“喂”数据，以及收集子进程的处理结果。

## 3. 基于 libevent 的异步事件驱动 I/O (thread.c)

**修正说明**：原结论中提到使用了 `libev`，实际上项目使用的是 **`libevent`** 库（通过 Makefile 中的 `-levent` 以及 `thread.c` 依赖 `<event.h>` 可知）。

* 为了高效处理大量数据和多个子进程的输入输出，避免 I/O 阻塞，项目启用了多线程 (`pthread`) 结合 **`libevent`** 事件循环。
* **Write 线程 (Write Event Thread)**：负责从主输入源不断读取数据，并按行异步地分发（Write）给所有子进程的输入管道。这里实现了一个状态机（`STAT_READ_MORE`, `STAT_WRITE_LINE`, `STAT_LAST_BUF`, `STAT_CLOSE_PIPE`）来处理数据的读取和分发逻辑。
* **Read 线程 (Read Event Thread)**：监听所有子进程的输出管道，一旦子进程有处理结果输出，就异步读取并将其写入到主程序的标准输出（STDOUT）中。

## 4. 自定义的高效缓冲管理 (iostream.c, tbuf.c)

* 实现了 `IOstream` 和 `TBuffer` 结构，提供了按行读取 (`get_line`)、尝试读取更多数据 (`try_read_more`) 以及动态扩容 (`expand_buffer`) 等功能，能够有效地处理超长文本行，防止管道 I/O 阻塞或内存越界。
* 使用了内存对齐分配 `posix_memalign` (`cmalloc` 函数)，以提高内存访问效率。

## 5. 信号处理与崩溃追踪 (main.c)

* **优雅退出**：拦截了 `SIGINT`, `SIGTERM`, `SIGQUIT` 等终止信号。当主进程收到这些信号时，会通过循环 `kill` 将信号传递给所有正在运行的子进程，确保不会产生孤儿进程。
* **忽略破裂管道**：忽略了 `SIGPIPE` 信号，防止因为部分管道破裂（如某个子进程意外退出）导致主程序崩溃退出。
* **崩溃堆栈追踪**：拦截了 `SIGSEGV`（段错误）信号，并在发生崩溃时使用 `backtrace` 和 `backtrace_symbols` 打印函数调用堆栈（Stack Trace），方便后续调试追踪。

## 总结

这是一个用于单机加速处理文本流的实用工具。如果你有一个极大的文本文件，需要逐行用某个脚本（如 awk, php, python 脚本）处理，单线程执行会非常慢。这个工具能帮你将这一过程并发化：自动启动指定数量的工作进程，自动将输入文件的内容按行非阻塞地分发给这些进程，并汇总打印它们的输出，从而极大提高单机文本处理的吞吐量。
